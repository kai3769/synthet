# Memory Consolidation

Goal: keep memory lean, accurate, and useful.

## Process

1. Read scars.md FIRST — check for patterns that apply
2. Read ALL memory .md files
3. Identify up to 5 improvement proposals:
   - *improve* — make entries more precise, concise, or better placed
   - *forget* — remove stale, outdated, or redundant information
   - *capture* — add new insights not yet in memory
4. For each proposal, self-evaluate:
   - *Obviously correct* → auto-apply silently
   - *Uncertain* → propose to user for approval
5. Commit and push auto-applied changes
