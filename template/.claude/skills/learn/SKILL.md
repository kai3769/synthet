# Learn

Extract patterns and principles from experience.

## Process

1. **Gather evidence** — recent conversations, scars, git log, issues
2. **Identify patterns:**
   - Repeated failures → new scar candidate
   - Approaches that work → principle candidate
   - User corrections → rule/preference update
   - Wasted effort → process improvement
3. **Extract conclusions** for each pattern:
   - What happened? (evidence)
   - Why? (root cause)
   - So what? (actionable conclusion)
   - Where does it go? (which file to update)
4. **Apply** — update the right memory files
5. **Commit and push**
